# E2E Test Model v2

## Overview

Документ описывает end-to-end тестирование приложения Origa на основе **реальных пользовательских сценариев** (User Journeys).

## Принципы

1. **Тесты создают данные** — каждый тест самостоятельно создаёт контент для проверки
2. **Изоляция** — тесты независимы и не влияют друг на друга
3. **Реальный путь** — тестируем то, как реально пользуется приложением человек

---

## User Journey 1: Полный цикл обучения (Primary)

**Цель:** Проверить основной сценарий использования приложения — от входа до завершения урока.

### Предусловия

- Зарегистрированный тестовый пользователь (email подтверждён)

### Шаги

| #   | Действие                                        | Ожидаемый результат                                    |
| --- | ----------------------------------------------- | ------------------------------------------------------ |
| 1   | Войти с валидными credentials                   | Редирект на `/home`, отображается HomePage             |
| 2   | Перейти на страницу кандзи `/kanji`             | Отображается пустой список кандзи                      |
| 3   | Нажать кнопку "+" для добавления кандзи         | Открывается модальное окно "Добавить кандзи"           |
| 4   | Выбрать уровень N5                              | Загружается список доступных кандзи N5                 |
| 5   | Выбрать несколько кандзи (например: 一, 二, 三) | Кандзи отмечены как выбранные                          |
| 6   | Нажать "Добавить"                               | Модальное окно закрывается, кандзи появляются в списке |
| 7   | Перейти на страницу урока `/lesson`             | Отображается первая карточка с кандзи                  |
| 8   | Нажать "Показать ответ"                         | Отображается ответ и кнопки оценки                     |
| 9   | Нажать "Знаю" (Good)                            | Переход к следующей карточке                           |
| 10  | Повторить для всех карточек                     | После последней — экран "Урок завершён!"               |
| 11  | Проверить статистику на экране завершения       | Отображается количество новых карточек                 |
| 12  | Нажать "На главную"                             | Редирект на `/home`                                    |
| 13  | Проверить, что счётчик кандзи обновился         | Счётчик показывает добавленные кандзи                  |

## User Journey 2: Добавление разных типов контента

**Цель:** Проверить, что пользователь может добавлять все типы обучающего контента.

### Предусловия

- Авторизованный пользователь с пустой коллекцией

### Шаги

| #   | Действие                             | Ожидаемый результат                           |
| --- | ------------------------------------ | --------------------------------------------- |
| 1   | Перейти на `/kanji`                  | Пустой список                                 |
| 2   | Добавить кандзи уровня N5 (一, 二)   | Кандзи отображаются в списке с фильтром "Все" |
| 3   | Перейти на `/words`                  | Пустой список                                 |
| 4   | Добавить слово "猫" (кошка)          | Слово отображается в списке                   |
| 5   | Добавить слово "犬" (собака)         | Оба слова отображаются                        |
| 6   | Перейти на `/grammar`                | Пустой список                                 |
| 7   | Добавить грамматику уровня N5        | Грамматические правила отображаются           |
| 8   | Проверить фильтры на каждой странице | Фильтры показывают корректные счётчики        |

## User Journey 3: Прохождение урока с разными оценками

**Цель:** Проверить SRS-систему и разные режимы оценки карточек.

### Предусловия

- Пользователь с добавленными кандзи/словами/грамматикой

### Шаги

| #   | Действие                                                  | Ожидаемый результат                |
| --- | --------------------------------------------------------- | ---------------------------------- |
| 1   | Перейти на `/lesson`                                      | Карточки загружены                 |
| 2   | Для первой карточки: "Показать ответ" → "Идеально" (Easy) | Переход к следующей                |
| 3   | Для второй карточки: "Показать ответ" → "Не знаю" (Again) | Переход к следующей                |
| 4   | Для третьей карточки: "Показать ответ" → "Плохо" (Hard)   | Переход к следующей                |
| 5   | Для четвёртой карточки: "Показать ответ" → "Знаю" (Good)  | Переход к следующей или завершение |
| 6   | Завершить все карточки                                    | Экран "Урок завершён!"             |
| 7   | Перейти в профиль → Проверить статистику                  | Статистика обновлена               |

## User Journey 4: Режим закрепления (Fixation)

**Цель:** Проверить режим закрепления сложных карточек.

### Предусловия

- Пользователь с карточками, оценёнными как "Не знаю" или "Плохо"

### Шаги

| #   | Действие                                                        | Ожидаемый результат                 |
| --- | --------------------------------------------------------------- | ----------------------------------- |
| 1   | Добавить кандзи и пройти урок с оценкой "Не знаю" для некоторых | Карточки отмечены как сложные       |
| 2   | Перейти на `/home`                                              | Отображается секция для закрепления |
| 3   | Нажать на закрепление (или перейти `/lesson?mode=fixation`)     | Загружаются только сложные карточки |
| 4   | Пройти закрепление                                              | Экран завершения                    |
| 5   | Проверить что карточки обновили статус                          | Сложность уменьшилась               |
