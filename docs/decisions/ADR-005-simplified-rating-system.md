# ADR-005: Simplified Rating System — 2-Button Review (Again/Good)

## Status

Accepted

## Date

2026-05-20

## Context

FSRS-5 поддерживает 4 рейтинга: Again / Hard / Good / Easy. В Anki 4 кнопки — стандарт. Но для изучения японского языка это создаёт избыточную когнитивную нагрузку.

### Проблемы четырёх кнопок

1. **Hard — самая misuse'имая кнопка** — повышает Difficulty (D) и замедляет рост Stability (S). Пользователи нажимают её вместо Again, искажая параметры FSRS.
2. **Easy — редко используется осознанно** — чаще по инерции или «на удачу», без реального понимания, зачем.
3. **Японский язык — бинарный** — либо вспомнил чтение/значение кандзи, либо нет. Промежуточные варианты не несут полезного сигнала.
4. **WaniKani, JPDB, Duolingo** — успешные приложения с 2 кнопками, подтверждающие, что бинарный выбор достаточен для массового пользователя.
5. **Автор FSRS (L.M.) подтвердил**: если пользователи misuse Hard, лучше его убрать — FSRS адаптируется.

## Decision

### Review UI: 2 кнопки

- «Don't know» → Again (Rating::Again)
- «Know» → Good (Rating::Good)

Hard и Easy **не отображаются** в обычном review-режиме.

### Маппинг Hard → Again

Неправильный ответ в YesNo/Quiz-режиме маппится на Again. Hard приравнивается к «не знаю».

### Easy сохранён для OnboardingScoring

`MarkCardAsKnownUseCase` (scoring_step.rs) использует Easy для отметки уже известных карт при онбординге. Это единственное место, где Rating::Easy передаётся в FSRS.

### Rating enum не меняется

Enum `Rating` остаётся с 4 вариантами — FSRS-5 требует все 4. Ограничение на 2 кнопки — только на уровне UI.

### Card history modal

4 цвета сохраняются для отображения прошлых рейтингов. Старые записи Hard/Easy отображаются корректно.

## Alternatives Considered

### Оставить 4 кнопки

Стандартный подход Anki, но с проблемой misuse Hard и повышенной когнитивной нагрузкой.

### 3 кнопки (убрать только Hard или только Easy)

Частичное решение. Не устраняет главную проблему — бинарность ответа при изучении японского.

## Consequences

- **Плюсы**: нулевая когнитивная нагрузка при выборе, невозможно misuse, мобайл-дружелюбно (крупные кнопки), бинарный ответ соответствует природе японского языка
- **Минусы**: тривиальные карты повторяются чуть чаще (Good вместо Easy ускорило бы рост S), FSRS получает менее гранулярный сигнал
- **Нет миграции данных**: Rating enum не меняется, старые Hard/Easy записи корректно отображаются в истории
