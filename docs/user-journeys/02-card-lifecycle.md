# User Journey: Card Lifecycle

## Цель

Пользователь исследует учебные материалы, создаёт карточки и управляет коллекцией.

---

## Phase 1: Discovery

### Step 1.1: Просмотр well-known sets

**Given:** Пользователь хочет посмотреть готовые наборы слов

**When:** Запрашивается список доступных сетов

**Then:**
- Возвращается список сетов с метаданными (id, type, level, title, description)
- Сеты включают JLPT N5-N1, Migii, Spy Family, Duolingo

---

### Step 1.2: Просмотр кандзи для уровня

**Given:** Пользователь хочет посмотреть кандзи для своего уровня

**When:** Запрашивается список кандзи для уровня (N5, N4, ...)

**Then:**
- Возвращается список кандзи с описанием, радикалами, чтениями, примерами слов
- Исключаются уже изученные кандзи (присутствующие в knowledge_set как Kanji cards с is_known)

---

### Step 1.3: Просмотр грамматики для уровня

**Given:** Пользователь хочет посмотреть грамматические правила

**When:** Запрашивается список правил для уровня

**Then:**
- Возвращается список правил с title, description, apply_to (части речи)
- Исключаются правила из existing_rule_ids

---

## Phase 2: Creation

### Step 2.1: Импорт well-known set

**Given:** Пользователь выбрал сет для импорта

**When:** Импортируется сет по id

**Then:**
- Для каждого слова из сета создаётся vocabulary card
- Перевод берётся из словаря
- Возвращается статистика: total_created_count, skipped_words, errors
- Сет помечается как импортированный

---

### Step 2.2: Создание vocabulary card вручную

**Given:** Пользователь хочет добавить слово

**When:** Вводится слово на японском

**Then:**
- Текст токенизируется
- Для каждого vocabulary token создаётся карточка
- Перевод берётся из словаря по native_language
- Карточка добавляется в knowledge_set

---

### Step 2.3: Создание kanji card

**Given:** Пользователь хочет добавить кандзи

**When:** Вводится список кандзи

**Then:**
- Для каждого кандзи создаётся карточка с описанием из словаря
- Карточка добавляется в knowledge_set

---

### Step 2.4: Создание grammar card

**Given:** Пользователь хочет добавить грамматическое правило

**When:** Вводится список rule_id

**Then:**
- Правило находится по id
- Создаётся карточка с содержимым правила
- Карточка добавляется в knowledge_set

---

## Phase 3: Management

### Step 3.1: Просмотр коллекции

**Given:** У пользователя есть карточки

**When:** Запрашивается список всех карточек

**Then:**
- Возвращается список всех StudyCard из knowledge_set

---

### Step 3.2: Добавление в избранное

**Given:** У пользователя есть карточка

**When:** Переключается is_favorite

**Then:**
- Флаг is_favorite инвертируется
- Возвращается новое значение

---

### Step 3.3: Удаление карточки

**Given:** У пользователя есть карточка

**When:** Удаляется карточка по card_id

**Then:**
- Карточка удалена из knowledge_set

---

## Edge Cases

### DuplicateCard
- **When:** Создаётся карточка с тем же question (word/kanji/title)
- **Then:** Ошибка DuplicateCard или пропуск (для batch операций)

### CardNotFound при Delete
- **When:** Удаляется несуществующая карточка
- **Then:** Ошибка CardNotFound

### CardNotFound при ToggleFavorite
- **When:** Переключается favorite у несуществующей карточки
- **Then:** Ошибка CardNotFound

### SetAlreadyImported
- **When:** Импортируется уже импортированный сет
- **Then:** Слова пропускаются как дубликаты
